# NOVA Specification and Architecture Repository

This repository contains the canonical, specification-first design definitions for the **NOVA Autonomous Productivity Operator**. It serves as the single source of truth for the conceptual architecture, ontology models, runtime specs, standards, and guides for the system.

## Directory Structure

* **[NAS/](file:///Users/sohamdhande/Docs_Local/NOVA/nova-docs/NAS)**: NOVA Architecture Specification. Defines *what* NOVA is conceptually. Organized into Volumes I through V. Immutable except via accepted RFCs.
* **[RFC/](file:///Users/sohamdhande/Docs_Local/NOVA/nova-docs/RFC)**: Request for Comments. The proposal mechanism for introducing or altering specifications.
* **[ADR/](file:///Users/sohamdhande/Docs_Local/NOVA/nova-docs/ADR)**: Architecture Decision Records. Documents technology choices and implementation reasoning without changing the core NAS.
* **[Research/](file:///Users/sohamdhande/Docs_Local/NOVA/nova-docs/Research)**: Background research, benchmarks, and theoretical justifications.
* **[Models/](file:///Users/sohamdhande/Docs_Local/NOVA/nova-docs/Models)**: Mathematical, ontological, and state-machine formalisms.
* **[Architecture/](file:///Users/sohamdhande/Docs_Local/NOVA/nova-docs/Architecture)**: Diagrammatic and visual system layouts.
* **[Standards/](file:///Users/sohamdhande/Docs_Local/NOVA/nova-docs/Standards)**: Project conventions, terminology, and naming schemas.
* **[Guides/](file:///Users/sohamdhande/Docs_Local/NOVA/nova-docs/Guides)**: Developer and contributor documentation.
* **[Roadmap/](file:///Users/sohamdhande/Docs_Local/NOVA/nova-docs/Roadmap)**: Incremental execution phases.

## Principles of the Repository

1. **Specification-First**: Write design specifications before code.
2. **Immutable History**: Never overwrite decisions or outcomes; document supercession instead.
3. **No Duplicate Prose**: Do not replicate definitions across files. Cross-reference using markdown links instead.
4. **Strict Formats**: Every document type must conform to its corresponding template schema.
